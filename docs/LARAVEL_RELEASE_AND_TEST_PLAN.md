# План Релизов И Тестирования Laravel

Дата: 2026-04-02  
Область: `PhpAssistent` (Laravel-сервер)

## Обязательная база

1. Семантическое версионирование:
- версия хранится в корневом файле `VERSION`;
- формат: `MAJOR.MINOR.PATCH`.

2. Дисциплина changelog:
- каждый релиз фиксируется в `CHANGELOG.md`;
- секция релиза должна соответствовать `VERSION` (`[X.Y.Z]`).

3. CI для каждого PR и push:
- установка зависимостей;
- проверки синтаксиса/формата;
- автоматические тесты;
- проверки релиза.

## Текущий релизный процесс

1. Обновить `VERSION`.
2. Добавить секцию релиза в `CHANGELOG.md`.
3. Запустить локальные проверки:
- `powershell -ExecutionPolicy Bypass -File scripts/release_check.ps1`
- `php artisan test`
4. Сделать commit и merge.
5. Создать тег: `vX.Y.Z`.

## Рекомендуемая стратегия тестирования

1. Unit-тесты:
- сервисы (`AppManager`, `DevicesReqest`, логика AES-оберток);
- поведение хелперов ролей/доступов.

2. Feature-тесты:
- auth/register/login;
- маршруты только для админа (widgets, apps, accounts);
- потоки установки приложений (валидация manifest/ошибки).

3. Регрессионные тесты:
- переключение темы/локали;
- поведение встраивания app-пунктов в меню.

## CI-гейты, которые нужно сохранить

- валидация формата `VERSION`;
- наличие секции текущей версии в `CHANGELOG`;
- `php -l` для критичных PHP-файлов (опциональный быстрый lint);
- успешное прохождение `php artisan test`.
